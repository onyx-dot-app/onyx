# Requirements Management with uv

This directory is kept for backwards compatibility with existing Docker builds.

## Overview

We use **`pyproject.toml`** as the single source of truth for all dependencies, with a unified **`uv.lock`** file for resolved versions.

### Why this approach?

- ✅ **Single source of truth**: All dependencies defined in `pyproject.toml`
- ✅ **No duplication**: Dependencies shared across environments are only listed once
- ✅ **Unified lock file**: All versions resolved together - guaranteed compatible
- ✅ **Fast**: `uv` is 10-100x faster than pip-tools
- ✅ **Reproducible builds**: Lock file pins all transitive dependencies
- ✅ **Easy updates**: Change `pyproject.toml`, run `uv lock`, done!

## File Structure

```
backend/
├── pyproject.toml              # SOURCE OF TRUTH - edit this!
├── uv.lock                     # Unified lock file (all versions)
├── compile_requirements.sh     # Script to regenerate uv.lock
└── requirements/               # Legacy .txt files (for Docker compat)
    ├── default.txt
    ├── dev.txt
    ├── ee.txt
    ├── model_server.txt
    └── combined.txt
```

## Workflow

### 1. Installing uv

If you don't have `uv` installed:

```bash
# On macOS/Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Or with pip
pip install uv
```

### 2. Adding/Updating Dependencies

**DO NOT** edit the `.txt` files directly! Instead:

1. Edit `pyproject.toml`
2. Add/update/remove dependencies in the appropriate section:
   - `[project.dependencies]` for **shared** dependencies (used by both backend and model_server)
   - `[project.optional-dependencies.backend]` for backend-only dependencies
   - `[project.optional-dependencies.model_server]` for model_server-only dependencies (ML packages)
   - `[project.optional-dependencies.dev]` for dev tools
   - `[project.optional-dependencies.ee]` for EE features
3. Regenerate lock file and requirements (see below)

### 3. Generating Lock File

```bash
./compile_requirements.sh
```

Or manually:
```bash
cd backend
uv lock
```

This resolves all dependencies (core + all extras) together into a single `uv.lock` file, ensuring all versions are compatible.

### 4. Installing Dependencies

```bash
# For backend production (shared + backend dependencies)
uv sync --extra backend

# For backend development (shared + backend + dev tools)
uv sync --extra backend --extra dev

# For backend with EE (shared + backend + ee)
uv sync --extra backend --extra ee

# For model server (shared + model_server, NO backend deps!)
uv sync --extra model_server

# For everything (testing)
uv sync --all-extras
```

**Key insight**: Model server only gets shared + ML packages, not the 100+ backend packages!

**Note**: All installations use the same `uv.lock` file, so versions are guaranteed compatible.

### 5. Upgrading Dependencies

To upgrade all dependencies to their latest compatible versions:

```bash
uv lock --upgrade
```

Or upgrade specific packages:

```bash
# Edit version in pyproject.toml, then:
uv lock
```

**Review changes carefully before committing!**

## Common Tasks

### Add a new dependency
```bash
# 1. Edit pyproject.toml, add to [project.dependencies]
vim pyproject.toml

# 2. Regenerate lock file
uv lock

# 3. Install
uv sync
```

### Update a single dependency
```bash
# 1. Edit version in pyproject.toml
vim pyproject.toml

# 2. Regenerate lock file
uv lock

# 3. Install
uv sync
```

### Check what changed
```bash
git diff uv.lock
```

## CI/CD Integration

In your CI pipeline:

```bash
# Install from lock file
uv sync --extra dev  # or appropriate extra

# Verify lock file is up to date (optional)
uv lock --check
```

### Docker Integration

For Docker builds that still use `requirements/*.txt` files, the compile script automatically exports from `uv.lock`:

```bash
# These are automatically generated by compile_requirements.sh:
# - requirements/default.txt      -> shared + backend
# - requirements/dev.txt          -> shared + backend + dev
# - requirements/ee.txt           -> shared + backend + ee  
# - requirements/model_server.txt -> shared + model_server (no backend!)
# - requirements/combined.txt     -> everything
```

This ensures Docker uses the exact same versions as `uv.lock`.

## Troubleshooting

### Conflict Resolution

If `uv lock` reports conflicts:

1. Check the error message for conflicting packages
2. Adjust version constraints in `pyproject.toml`
3. Run `uv lock` again

### Platform-Specific Dependencies

`uv.lock` is cross-platform by default and includes markers for platform-specific packages. It will work on Linux, macOS, and Windows.

## Migration Notes

This setup was migrated from individual pinned requirements files. The key changes:

- **Before**: Multiple `.txt` files with duplicated dependencies
- **After**: Single `pyproject.toml` source of truth, compiled `.txt` lock files

All duplicates have been removed, with shared dependencies moved to `[project.dependencies]`.

For more info on `uv`: https://github.com/astral-sh/uv
