# Glossary

Terms used across this PoC's docs, code, and demos that aren't immediately
obvious to a general software engineer. Alphabetical. Networking-specialist
jargon gets a sentence about *why it matters here*, not just what it means in
the abstract.

| Term | Meaning |
|------|---------|
| **AES-256-CBC** | A symmetric encryption mode: AES with a 256-bit key, in Cipher Block Chaining mode. Used here (with PBKDF2) to encrypt the CA private key at rest. The cipher is fine; modern preference is AES-GCM (authenticated), but CBC matches `openssl enc` defaults and round-trips cleanly between the host (gen-ca.sh) and the proxy. |
| **ALPN (Application-Layer Protocol Negotiation)** | A TLS extension where client and server agree on the application protocol (HTTP/1.1, HTTP/2, h3) during the TLS handshake itself. mitmproxy needs to mirror ALPN correctly so the proxied request uses the protocol the client expected. |
| **Audit event** | One JSON object emitted to stdout per lifecycle transition (received, authenticated, policy_decision, injected, forwarded, responded, denied, error). The proxy's only persistence mechanism in v0. |
| **Broker** | The credential-policy service. The *only* component that reads `secrets/`. The proxy calls `POST /policy/evaluate` with `{session_token, method, host, path}` and gets back `{decision, inject_headers, strip_headers, upstream_url}`. Single-purpose by design so it can be locked down independently. |
| **CA (Certificate Authority)** | The trust anchor whose signature on a certificate makes that certificate trusted. We mint our own private CA (the Onyx CA), install it in every sandbox's trust store, and use it to sign the per-host leaf certs mitmproxy generates on the fly. |
| **CAfile / CApath** | Two ways curl (via OpenSSL) finds trusted CAs: `CAfile` is one PEM file, `CApath` is a directory of hashed cert files. They're additive — setting `--cacert` doesn't disable `--capath`. Demo 04 has to override both to genuinely pin a client to one CA. |
| **Calico / Cilium** | Production-grade Kubernetes CNIs (see CNI) that enforce NetworkPolicy. Either is the right choice for V1; `kind`'s default kindnet doesn't enforce policy, so to exercise L1 in explicit mode in a local cluster you'd swap kindnet out. |
| **CapDrop / Linux capability** | A finer-grained alternative to root/non-root. Capabilities (`NET_ADMIN`, `SYS_ADMIN`, `CHOWN`, etc.) are individual privileges that can be granted or dropped per-process. `cap_drop: [ALL]` strips everything; the workload can't modify firewall rules even running as UID 0. |
| **ClusterIP (Kubernetes Service)** | The stable virtual IP a Kubernetes Service publishes; kube-proxy programs iptables (or IPVS) on every node so traffic to that IP gets DNAT'd to one of the backing pods. The K8s analog of a docker-compose static IP on a bridge. |
| **CNI (Container Network Interface)** | The plugin contract Kubernetes uses to set up pod networking. The CNI chosen by the cluster decides how pods get IPs, how pod-to-pod traffic flows, and whether `NetworkPolicy` is enforced. kindnet (kind's default) doesn't enforce policy; Calico, Cilium, AWS VPC CNI do. |
| **Certificate pinning** | A client hardcoding which exact CA(s) or certificate(s) it will accept, refusing all others including ours. Defeats MITM by design — there is no proxy-side fix. Demo 04 exercises this; we fail-closed (block), never tunnel-around. |
| **certifi** | Python package that ships its own bundled CA list (`cacert.pem`). `requests`, `httpx`, `urllib3`, `aiohttp` all use this *instead of* the OS trust store unless explicitly overridden. Patching certifi is the only way to make Python HTTP clients trust the Onyx CA without changing user code. |
| **ClientHello** | The first message of a TLS handshake, sent by the client. Contains the SNI (the requested hostname), supported cipher suites, ALPN preferences, etc. All unencrypted — anyone on the wire (including our proxy) can read it. |
| **conntrack** | Linux's connection-tracking subsystem. When iptables DNATs a packet, conntrack remembers the rewrite so reply packets get the reverse rewrite. Without conntrack, the application would see responses coming from the rewritten IP instead of the original — and TCP would break. |
| **CONNECT (HTTP method)** | The method an HTTP proxy uses to establish a TCP tunnel to an arbitrary host:port. Used for HTTPS-through-proxy: client sends `CONNECT api.allowed.example:443 HTTP/1.1`, proxy returns `200`, client does its TLS handshake through the now-opaque tunnel. mitmproxy intercepts at the TLS layer inside that tunnel. |
| **DNAT (Destination NAT)** | An iptables action that rewrites a packet's destination IP/port. Our transparent mode rule `iptables -t nat -A OUTPUT -p tcp --dport 443 -j DNAT --to-destination 172.30.0.10:8443` redirects any port-443 traffic to the proxy regardless of the original destination. The app is unaware. |
| **DROP vs REJECT (iptables targets)** | Both block a packet. DROP silently discards (sender sees a timeout). REJECT sends an explicit error back (sender sees connection refused / network unreachable, fails fast). We use REJECT so misbehaving code fails immediately rather than hanging. |
| **ECH (Encrypted Client Hello)** | A newer TLS extension that encrypts the SNI itself. Not yet ubiquitous; doesn't affect this PoC (the proxy IS the on-path entity, so cleartext SNI is fine for us). Mentioned in the SNI glossary entry for completeness. |
| **Egress** | Outbound network traffic — connections leaving the sandbox toward other services or the internet. This whole PoC is "egress interception." |
| **ENOENT ("No such file or directory")** | A standard Linux error code. Python surfaces it as `FileNotFoundError`. We hit it specifically from `getsockopt(SO_ORIGINAL_DST)` in the K8s port: the kernel reports "no conntrack entry for this socket" by raising ENOENT, which mitmproxy treats as fatal until we patch around it (see `proxy/server.py`). |
| **Explicit proxy** | A proxy the client is *configured* to use, via env vars like `HTTPS_PROXY`. Cooperative; relies on the client honoring its config. Our Variant A. Contrast with transparent. |
| **extra_hosts** | A Docker Compose field that adds `/etc/hosts` entries to a container. We use it to give mock hostnames (`api.allowed.example`) a deterministic resolvable IP (in TEST-NET-3) so demos work without real DNS. |
| **Fail-closed / fail-open** | On error or uncertainty, fail-closed denies (the safe choice for security policy); fail-open permits (the safe choice for availability). The PoC is fail-closed everywhere: missing SNI, broker timeout, cert pinning, invalid host all result in a block. |
| **Fernet** | A symmetric-encryption token format (base64-encoded ciphertext+HMAC+timestamp) from Python's `cryptography` library. The plan calls for it, but the implementation uses `openssl enc -aes-256-cbc -pbkdf2` instead so the host doesn't need a Python crypto dep — same security properties, different framing. |
| **Firecracker** | Amazon's microVM hypervisor used for sandbox isolation (Lambda, Fargate). Stronger isolation boundary than containers. Orthogonal to this PoC; mentioned in known-gaps as a separate concern. |
| **getaddrinfo (libc)** | The standard C library function for DNS resolution. The Anthropic SOCKS5 CVE turned on this: JS `endsWith('.google.com')` saw one hostname, `getaddrinfo()` C-string-truncated on `\x00` and saw another — allowlist bypass via `evil.com\x00.google.com`. The PoC's `validators.py` rejects nulls and control chars to prevent this class. |
| **gVisor** | Google's user-space kernel for sandboxed containers. Like Firecracker, a stronger isolation layer than vanilla containers. Out of scope here. |
| **HTTP/2, HTTP/3, QUIC** | Newer HTTP versions. HTTP/3 runs over QUIC (a UDP-based transport). We reject all UDP except DNS in transparent mode, so HTTP/3 is unsupported — documented in known-gaps. |
| **HTTPS_PROXY / https_proxy / HTTP_PROXY / NO_PROXY** | The de facto convention for explicit-proxy configuration via env vars. Most HTTP clients (curl, requests, httpx, urllib, Go's stdlib) honor these. We set ~10 such vars in compose (uppercase and lowercase variants) — matching Anthropic Managed Agents' "set ten env vars" pattern. |
| **hostAliases (Kubernetes)** | A pod-spec field that adds entries to the pod's `/etc/hosts`. The K8s equivalent of docker-compose's `extra_hosts` — same role here (pinning the mock hostnames to TEST-NET-3 IPs). |
| **IDN (Internationalized Domain Name) / homograph** | A domain that uses non-ASCII characters (Unicode), or one that looks visually identical to an ASCII domain (е.g. Cyrillic `а` vs Latin `a` in `аpple.com`). A parser-differential trap: one component may match the visual form, another the Punycode. The PoC rejects non-ASCII hosts entirely in v0. |
| **Init container** | A container that runs to completion *before* the main container starts, typically with elevated privileges. We use one (`iptables-init`) to install firewall rules with `NET_ADMIN`; the workload itself gets no `NET_ADMIN`. Same pattern Kubernetes uses everywhere. |
| **internal: true (Docker network)** | A Docker network option that disables NAT to the host. Containers on the network can talk to each other but have no route to the outside world. This is L1 in our defense model — the workload has nowhere to send raw TCP except the proxy. |
| **iptables** | The Linux kernel firewall configuration tool. Operates on packet-processing chains (PREROUTING, INPUT, OUTPUT, POSTROUTING, FORWARD) in tables (filter, nat, mangle, raw). We use OUTPUT chain in the nat table for DNAT, and OUTPUT in the filter table for REJECT. |
| **ip6tables** | The IPv4-only sibling. `iptables` doesn't touch IPv6; you need `ip6tables` separately. We block IPv6 wholesale (sysctl + `ip6tables -P OUTPUT DROP`) rather than trying to mirror the v4 rules. |
| **kind (Kubernetes IN Docker)** | A tool that runs a real Kubernetes control plane and nodes inside Docker containers. Used here for local-only cluster testing. `kind load docker-image` ships a locally-built image into the cluster's containerd without going through a registry. |
| **kindnet** | `kind`'s default CNI plugin. Minimal, fast, and **does not enforce NetworkPolicy** — a known limitation that affects this PoC's explicit-mode L1 demo under kind. Production clusters use a policy-enforcing CNI (see Calico / Cilium). |
| **KMS (Key Management Service)** | A managed-service abstraction for storing and using encryption keys (AWS KMS, GCP KMS, HashiCorp Vault). V1 will source `ENCRYPTION_KEY_SECRET` from KMS instead of a compose env var. |
| **kube-proxy** | The Kubernetes component that, on every node, programs iptables (or IPVS) to make Service ClusterIPs route to backing pods. When the sandbox connects to the proxy's Service IP, two DNATs happen back-to-back: workload netns rewrites to the Service IP, host netns rewrites again to a pod IP. |
| **L1 / L2 / L3 (layered defense)** | This PoC's shorthand for the three enforcement layers: L1 TCP-level isolation (`internal: true` / NetworkPolicy), L2 HTTP-steering (`HTTPS_PROXY` or iptables DNAT), L3 TLS MITM + policy enforcement (mitmproxy + addon stack). NOT the OSI model — borrowing only the numeric ordering. |
| **Leaf certificate** | The end-entity certificate that a TLS server presents to a client, signed (directly or via intermediates) by a trusted CA. mitmproxy mints a fresh leaf per hostname on the fly, signed by the Onyx CA. The sandbox sees `subject: CN=api.allowed.example; issuer: O=Onyx; CN=Onyx Egress PoC TLS Inspection CA`. |
| **MITM (man-in-the-middle)** | An on-path entity that terminates the client's TLS, decrypts traffic, inspects/modifies it, then makes its own connection onward. Normally adversarial. Here it's a deliberate, friendly interception by infrastructure we own. The "M" is the whole point. |
| **mitmproxy** | The open-source MITM-as-a-library project. We use it (not the CLI) because of its rich addon hook surface (`tls_clienthello`, `http_connect`, `request`, `response`), on-the-fly cert generation, leaf-cert caching, and HTTP/2 / ALPN / CONNECT handling. Pinned to 10.x. |
| **mTLS (mutual TLS)** | TLS where both sides authenticate via certificates. Stripe Smokescreen uses mTLS for client identity. We don't — our explicit-mode auth is `Proxy-Authorization: Basic`, our transparent-mode auth is source-IP/pod-identity (V1). |
| **NAT (Network Address Translation)** | Rewriting IP addresses (and/or ports) on packets in flight. DNAT rewrites destination; SNAT rewrites source. Docker bridges use SNAT to give containers internet access; we use DNAT to redirect container traffic to the proxy. |
| **NET_ADMIN** | The Linux capability required to modify network configuration (including iptables rules). The init container has it; the workload doesn't. That asymmetry is what makes the rules tamper-resistant from inside. Some managed Kubernetes clusters (EKS Fargate, GKE Autopilot) forbid `NET_ADMIN` entirely — they can't run transparent mode at all. |
| **NetworkPolicy (Kubernetes)** | The K8s equivalent of Docker's `internal: true` — declarative rules about which pods can talk to which. V1 will use NetworkPolicy to enforce L1 isolation. |
| **Network namespace (netns)** | The Linux kernel's per-process network-stack isolation primitive: separate interfaces, routes, iptables tables, etc. Containers each get their own. The `iptables-init` container shares the workload's netns (via `network_mode: "service:..."`), which is how rules applied in one container govern the other. |
| **NODE_EXTRA_CA_CERTS** | Node.js's mechanism for adding a CA to its trust store at process startup. Node ships its own CA list and ignores both certifi and the OS trust store; this env var is the only way to extend that. |
| **OUTPUT chain (netfilter)** | The netfilter chain that processes packets originating from the local host. We install DNAT here so locally-originated traffic from the workload is rewritten before it leaves. Distinct from PREROUTING (packets arriving from elsewhere) and POSTROUTING (packets leaving after routing). |
| **Parser-differential bug** | A vulnerability where two parsers in a system disagree on the meaning of an input, and the disagreement is security-relevant. Anthropic's SOCKS5 CVE: JS `endsWith` saw one hostname, libc `getaddrinfo` saw another. Defense: canonicalize the input at one boundary and use the canonical form everywhere. |
| **PBKDF2 (Password-Based Key Derivation Function 2)** | Standard for turning a passphrase into an encryption key, by salted SHA-256 hashing repeated many times (we use 100k iterations). Adds CPU cost to brute-force attacks. Used by `openssl enc -pbkdf2`. |
| **PEM** | The text-encoded format for cryptographic objects (`-----BEGIN CERTIFICATE-----` ... `-----END CERTIFICATE-----`). Base64-wrapped DER. The native serialization for openssl and mitmproxy. |
| **POSTROUTING / PREROUTING (netfilter)** | The other two iptables chains we don't use. PREROUTING runs on incoming packets before routing; POSTROUTING runs on outgoing packets after routing. We do all our work in OUTPUT (locally-originated, before routing). |
| **Proxy-Authorization** | The HTTP header used for HTTP-proxy authentication. Carries `Basic base64(user:pass)`. We use `Basic base64(session:<session_token>)`. Standard mechanism; every HTTP client populates it automatically when given `http://user:pass@proxy:port`. Stripped from the request before forwarding upstream. |
| **REJECT vs DROP** | See DROP. |
| **REQUESTS_CA_BUNDLE / SSL_CERT_FILE / CURL_CA_BUNDLE / etc.** | Per-library env vars overriding the trust-store path. Each Python/Go/Node/Java library reads a different one. There's no single "trust the Onyx CA" knob — you set all of them, install into the OS store, *and* patch certifi to be sure. |
| **RFC 5737 / TEST-NET-3** | A reserved IPv4 range (`198.51.100.0/24`) the IETF set aside for documentation and examples. Guaranteed to never route on the real internet. We use it as the IP for mock hostnames so demos can't accidentally hit a real server. |
| **RSA** | The asymmetric encryption algorithm we use for the CA keypair. 4096-bit because CAs live for years and key length matters at that timescale. Modern alternatives (ECDSA, Ed25519) are faster but RSA is universally supported by every HTTP client and CA-chain validator. |
| **Sidecar** | A container running alongside the main one in the same pod, sharing its network namespace but typically not its filesystem or process list. The `iptables-init` container is a degenerate sidecar — it shares netns and exits immediately. |
| **SNAT (Source NAT)** | Rewriting the source IP of outgoing packets. Docker bridges do this for outbound traffic so containers appear to come from the host's IP. Not used here; we DNAT, not SNAT. |
| **SNI (Server Name Indication)** | A TLS extension where the client states the destination hostname in the ClientHello, in cleartext. Lets one IP host many TLS sites (CDNs, shared hosting). mitmproxy reads SNI to know which leaf cert to mint. The PoC fails closed on missing SNI — there's no safe default for "I don't know which destination this is." |
| **SO_ORIGINAL_DST** | A Linux socket option that asks "before iptables DNAT rewrote this packet's destination, what was it?" mitmproxy uses this in transparent mode to recover the original destination IP:port. Linux-specific; the whole transparent-proxy mechanism depends on it. |
| **SOCKS5** | An older proxy protocol (RFC 1928). Anthropic's local sandbox-runtime had two SOCKS5 CVEs (null-byte parser-differential, allowlist bypass). We use HTTP CONNECT instead of SOCKS5; the parser-differential lesson still applies. |
| **SPIFFE** | A standard for workload identity (X.509 certs + URIs). One possible mechanism for source-identity-based auth in transparent mode in V1. |
| **SSRF (Server-Side Request Forgery)** | The vulnerability class where a server can be tricked into making requests to internal services it shouldn't reach (cloud metadata endpoints, internal admin APIs). Stripe's Smokescreen exists to prevent SSRF — different problem from credential mediation. Mentioned in the industry survey. |
| **TLS (Transport Layer Security)** | The encryption protocol behind HTTPS. Encrypts headers and body end-to-end between client and server. The only way for an on-path entity to read or modify TLS traffic is MITM (terminate + decrypt + re-encrypt). |
| **TLS termination** | The act of decrypting TLS at a non-endpoint (the proxy). After termination, the proxy has cleartext bytes; it can inspect, modify, inject headers, then re-encrypt (or not) on the way out. |
| **TPROXY** | A Linux netfilter target similar to REDIRECT but preserving the original destination IP without conntrack. Used by Cloudflare's transparent-proxy stack. The PoC uses REDIRECT/DNAT; TPROXY would be the alternative. |
| **Transparent proxy** | A proxy that intercepts traffic at the network layer; the application is unaware and needs no configuration. Our Variant B, via iptables DNAT. |
| **Trust store** | The set of CA certificates a system or library trusts when validating a server's cert. We need ours installed in *every* trust store the sandbox's HTTP clients consult — OS bundle (curl, git), certifi (Python HTTP libs), Node bundle (`NODE_EXTRA_CA_CERTS`), Java cacerts if present. |
| **update-ca-certificates** | The Debian/Ubuntu command that scans `/usr/local/share/ca-certificates/` and rebuilds the system trust bundle at `/etc/ssl/certs/ca-certificates.crt`. We invoke it once during image build after dropping `ca.crt` into the source dir. |
| **WebSocket upgrade** | A mechanism for switching an HTTP/1.1 connection to a long-lived bidirectional WebSocket session. mitmproxy handles this transparently. The PoC doesn't have demos for it; out of scope for v0. |
